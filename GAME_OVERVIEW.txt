LINE SLICE BALLS
GAME OVERVIEW (CURRENT BUILD)

1) PROJECT SUMMARY
- Platform: Web, mobile-first portrait layout
- Stack: HTML + CSS + JavaScript
- Renderer: Canvas 2D
- Runtime entry: game.js loaded by index.html
- Mode: Endless single-player arcade


2) CORE GAME LOOP
1. Attacker starts attached to a wall.
2. Opening tutorial line shows the intended slice path from the attacker.
3. Survival timer does not start until the player's first real dash.
4. Player swipes to choose a wall-to-wall dash path.
5. If dash path intersects the moving defender line, it counts as a slice.
6. Score, combo, coins, and timer update.
7. Defenders and hazards continue moving.
8. Run ends when timer reaches 0.


3) MOVEMENT MODEL
- Attacker:
  - Wall-attached while idle.
  - No free movement inside the arena.
  - Very fast dash travel.
  - Dash duration is short (~0.08 to 0.14 sec).
  - Post-dash attach pulse on wall impact.
- Dash cooldown:
  - 0.15 sec safety cooldown between dashes.
- Dash target:
  - Always resolves to a wall landing point based on swipe direction.


4) INPUT / AIM / TUTORIAL
- Touch or swipe can begin from anywhere in the arena.
- Aim preview shows:
  - full projected dash path
  - wall landing marker
- Opening tutorial guide:
  - starts from the attacker
  - points through the defender line toward a valid slice path
  - stays visible until the first actual launch


5) SLICE DETECTION
- Slice is path-based:
  - If the dash segment intersects the defender segment, it is a slice.
  - Dynamic detection is used while defenders move.
- Valid slice takes priority over defender-ball bounce:
  - if the dash angle would pass through the line and slice it, the slice is awarded
  - defender-ball bounce only triggers when the dash does not already qualify as a slice
- Slice reward:
  - +1 score
  - +1 coin
  - combo increment
  - timer reset to 3.0 seconds


6) DEFENDERS / LINE
- Two defender balls move continuously through the arena.
- Defender balls enforce a minimum separation distance.
- Defender line visuals:
  - straight by default
  - vibrates / flashes on successful slice
- Defender-ball contact:
  - can bounce the attacker back to the launch wall if no valid slice occurred


7) TIMER / COMBO / COINS
- No heart system.
- Survival rule:
  - player must land at least 1 slice every 3.0 seconds
  - timer starts only after the first dash
  - timer reaching 0 immediately triggers game over
- Low-time feedback:
  - map visually heats up when close to timeout
- Miss:
  - no point is awarded
  - combo resets to 0
  - MISS banner + red flash
- Coins:
  - 1 coin per slice
  - persisted in localStorage
- Record:
  - best score is saved in localStorage
  - shown in menu and game-over screen


8) HAZARDS
- Spikes:
  - 1st spike unlocks at 5 slices
  - 2nd spike unlocks at 30 slices
  - spikes move across the arena
  - hitting a spike bounces attacker back and invalidates the dash
- Arrows:
  - unlock at 50 slices
  - warning lane appears before spawn
  - arrow hit counts as a miss


9) BOUNCE RULES
- Spike contact:
  - attacker rebounds back toward the launch wall
  - dash does not count as a slice
- Defender-ball contact:
  - attacker rebounds back only if no valid line slice occurred first
- Arrow contact:
  - no rebound logic, hit counts as a miss


10) ATTACKER SKINS (COSMETIC)
- Character silhouette:
  - round attacker ball
  - thick outline + glow aura
- Skin roster:
  - Neon Ninja: default unlocked
  - Fire Ninja: 150 coins
  - Water Ninja: 150 coins
  - Lightning Ninja: 250 coins
  - Earth Ninja: 250 coins
  - Blossom Ninja: 300 coins
- Skin effects are cosmetic only.

  - themed particles
  - meteor-style movement flare
  - afterimages

  - spark burst
  - line flash / vibration
  - warped pop-up banner
  - screen flash + optional shake
- Arena:
  - bright near-white playable map interior
  - warm framed outer shell
  - low-time heat shimmer effect
- Tutorial:
  - animated slice-path guide from attacker to the line


12) UI / OVERLAYS

  - timer bar

- Overlays:
  - Main menu
