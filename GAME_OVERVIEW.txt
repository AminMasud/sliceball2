LINE SLICE BALLS
GAME OVERVIEW (DASH UPDATE)

1) PROJECT SUMMARY
- Platform: Web (single page, Canvas 2D)
- Stack: HTML + CSS + JavaScript
- Runtime entry: game.js loaded by index.html
- Mode: Endless single-player arcade


2) CORE GAME LOOP
1. Attacker is attached to a wall.
2. Player touches attacker and swipes.
3. Game calculates wall target from swipe direction.
4. Attacker performs a fast dash to that wall point.
5. Slice check happens during dash path.
6. Score/combo/coins/hearts update.
7. Defenders keep moving and retargeting continuously.
8. Repeat until hearts reach 0.


3) MOVEMENT MODEL (CURRENT)
- Attacker is wall-attached at all times when idle.
- Attacker can attach to top, bottom, left, or right wall.
- No free-rolling physics movement for attacker anymore.
- Dash behavior:
  - straight line
  - high-speed travel
  - short duration (~0.15 to 0.25 seconds)
  - post-dash wall stick
- Dash safety cooldown: 0.15 seconds.


4) AIM / INPUT
- Pointer/touch:
  - touch anywhere (including middle of arena)
  - swipe direction
  - release to dash
- Aim preview:
  - glowing path line
  - landing marker on wall


5) SLICE DETECTION
- Slice is checked along dash segment each frame.
- Rule requires true crossing behavior:
  - path intersects defender segment
  - side changes across defender line
  - touch-only contact does not count
- On slice:
  - +1 score
  - +1 coin
  - combo increment
  - line vibration + FX


6) DEFENDERS / LINE
- Two CPU defenders stay in upper half.
- Defenders move continuously (not per-turn waiting).
- Defender line rendering:
  - straight normally
  - zigzag only when hit (slice reaction window)


7) HEARTS / COMBO / COINS
- Hearts: max 2.
- Miss:
  - lose 1 heart
  - combo reset
  - red flash + MISS banner
- Heart restore:
  - every 5 successful slices
  - capped at 2
- Coins:
  - 1 per slice
  - persisted in localStorage


8) OBSTACLES
- Spiked balls unlock at 50 slices.
- Slow drifting/rotating behavior.
- Hitting a spike counts as a miss.


9) VISUAL FEEDBACK
- Dash:
  - bright trail segments
  - spark particles
  - afterimages
- Wall attach:
  - impact pulse ring
- Slice:
  - sparks burst
  - line vibration/flash
  - screen flash + optional shake
- Arena:
  - clean plain map color


10) UI / OVERLAYS
- HUD:
  - top-left hearts
  - top-center combo
  - top-right coins
- Overlays:
  - Main menu (Play / Shop / Options)
  - Shop (skin unlock/equip)
  - Options (screen shake toggle)
  - Game over (final score / best combo / coins earned)


11) FILE MAP (ACTIVE)
- index.html
  - canvas + endless HUD + overlays
- styles.css
  - visual theme + animations
- game.js
  - all active gameplay logic and rendering
- src/
  - legacy files from prior architecture (not active runtime path)


12) RUN
- Open index.html in browser.
- No backend required.
